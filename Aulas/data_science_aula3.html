Fundamento

Amostragem

A arte de representar o todo pela parte

🎯 Definição

Amostragem é o processo de seleção de um subconjunto representativo de uma população maior, permitindo inferências estatísticas confiáveis sem analisar todos os dados.

População vs Amostra

População: Conjunto completo de elementos de interesse
Amostra: Subconjunto selecionado da população
Objetivo: A amostra deve representar fidedignamente a população

Por que Amostrar?

  • Redução de custos computacionais
  • Velocidade de processamento
  • Viabilidade prática
  • Controle de qualidade

⚖️ Conceitos Fundamentais

Parâmetro: Característica da população (μ, σ)
Estatística: Característica da amostra (x̄, s)
Erro Amostral: Diferença entre estatística e parâmetro
Viés: Distorção sistemática na amostragem

Métodos

Tipos de Amostragem

Escolhendo a estratégia adequada

🎲 Aleatória Simples

Cada elemento tem igual probabilidade de seleção

Vantagem: Sem viés de seleção
Desvantagem: Pode não capturar subgrupos importantes

📊 Estratificada

População dividida em estratos homogêneos

Vantagem: Garante representação de subgrupos
Desvantagem: Requer conhecimento dos estratos

🎯 Sistemática

Seleção a intervalos regulares

Vantagem: Simples implementação
Desvantagem: Pode criar viés se há padrões nos dados

🎪 Por Conglomerados

Seleção de grupos inteiros

Vantagem: Eficiente para populações dispersas
Desvantagem: Menor precisão que outros métodos

🏪 Exemplo: E-commerce

Aleatória: Sortear 1000 clientes aleatoriamente
Estratificada: 200 clientes de cada região (5 regiões)
Sistemática: Cada 10º cliente da lista ordenada
Conglomerados: Selecionar 5 cidades e analisar todos os clientes

Cálculo

Determinando o Tamanho da Amostra

Equilibrando precisão e viabilidade

🎯 Fatores Determinantes

O tamanho ideal da amostra depende do nível de confiança desejado, margem de erro aceitável, variabilidade da população e recursos disponíveis.

📐 Fórmula Básica

n = (Z² × p × (1-p)) / E²

n: tamanho da amostra
Z: valor crítico (1.96 para 95%)
p: proporção estimada
E: margem de erro

🎪 Regras Práticas

  • 30+ para distribuição normal (TLC)
  • 100+ para proporções
  • 400+ para pesquisas nacionais
  • 1000+ para segmentações detalhadas

⚖️ Trade-offs

Amostra Maior:
✅ Menor erro amostral
✅ Maior confiança
❌ Maior custo
❌ Mais tempo

🚨 Cuidados Especiais

  • População finita: aplicar correção
  • Eventos raros: aumentar amostra
  • Análise por subgrupos: considerar separadamente
  • Taxa de resposta: ajustar para não-respostas

Distribuições

Distribuição de Dados

Compreendendo padrões nos dados

📊 Conceito

Distribuição de dados descreve como os valores estão espalhados ou concentrados, revelando padrões fundamentais para análise estatística e tomada de decisão.

🔔 Normal (Gaussiana)

Simétrica, em forma de sino

  • Média = Mediana = Moda
  • 68% dos dados em ±1σ
  • 95% dos dados em ±2σ
  • Base para muitos testes

📈 Assimétrica (Skewed)

Tendência para um lado

  • Positiva: cauda à direita
  • Negativa: cauda à esquerda
  • Comum em dados reais
  • Afeta interpretação de médias

📏 Uniforme

Todos os valores igualmente prováveis

  • Retangular
  • Variância máxima
  • Rara na natureza
  • Útil para simulações

📉 Exponencial

Decaimento exponencial

  • Eventos raros
  • Tempos entre falhas
  • Duração de chamadas
  • Altamente assimétrica

🎯 Teorema Central do Limite

Independente da distribuição original, a média amostral tende à distribuição normal com amostras grandes (n≥30). Este princípio fundamenta muitas análises estatísticas em Data Science.

Aplicação

Amostragem em Data Science

Cases práticos e estratégias

🤖 Machine Learning

  • Divisão treino/validação/teste
  • Cross-validation
  • Balanceamento de classes
  • Bootstrap sampling

📊 A/B Testing

  • Grupos de controle e tratamento
  • Randomização
  • Poder estatístico
  • Significância estatística

🔍 Análise Exploratória

  • Amostras para prototipagem
  • Teste de hipóteses
  • Validação de modelos
  • Detecção de outliers

⚡ Big Data

  • Reservoir sampling
  • Streaming data
  • Aproximações estatísticas
  • Sampling bias em algoritmos

🎯 Estratégia de Amostragem para ML

70/15/15: 70% treino, 15% validação, 15% teste
Estratificada: Manter proporção de classes
Temporal: Respeitar ordem cronológica em séries temporais

⚠️ Armadilhas Comuns

  • Viés de seleção: Amostra não representativa
  • Data leakage: Informação do futuro no treino
  • Overfitting: Modelo decorou a amostra
  • Underfitting: Amostra insuficiente para capturar padrões

💭 Desafios Reais

Problema: Sistema de recomendação com dados desbalanceados (95% produtos não comprados). Como amostrar para treinar o modelo?

Ética: Como garantir que nossa amostragem não introduz viés contra grupos minoritários?

Exemplo

Case: Análise de Vendas Online

Aplicando amostragem e distribuições

1. Contexto

E-commerce com 5 milhões de transações anuais, 50 mil produtos, 500 mil clientes em 5 regiões

2. Objetivo

Analisar padrões de compra para otimizar estoque e campanhas de marketing

3. Estratégia de Amostragem

Amostragem estratificada por região (20% de cada) e sistemática temporal (cada 5º dia)

4. Análise de Distribuições

Valores de pedido (log-normal), frequência de compra (exponencial), avaliações (normal)

📊 Descobertas

  • Valores seguem distribuição log-normal
  • 80% dos clientes compram ≤ 3x/ano
  • Sazonalidade clara em novembro/dezembro
  • Diferenças regionais significativas

⚡ Implicações

  • Segmentação por valor CLV
  • Campanhas de reativação
  • Planejamento sazonal
  • Personalização regional

🎯 Lições Aprendidas

A amostragem estratificada revelou padrões regionais que uma amostra aleatória simples poderia mascarar. A identificação da distribuição log-normal orientou o uso de métricas apropriadas (mediana vs. média).

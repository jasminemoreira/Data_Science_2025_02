Fundamentos

Evolução da Ciência de Dados

Da computação estatística à inteligência artificial

A ciência de dados emergiu como disciplina formal na década de 2000, mas suas raízes remontam à estatística, pesquisa operacional e ciência da computação. O termo "Data Science" foi popularizado por DJ Patil e Jeff Hammerbacher, considerados os primeiros Chief Data Officers de grandes corporações tecnológicas.

1960-1980: Era da Computação Estatística

Desenvolvimento de sistemas de banco de dados relacionais e primeiras aplicações de estatística computacional. Surgimento do SQL e sistemas de informação gerencial.

1990-2000: Business Intelligence

Consolidação de data warehouses, ferramentas de OLAP e primeiros sistemas de apoio à decisão. Início da mineração de dados comercial.

2000-2010: Web Analytics e Big Data

Explosão de dados web, surgimento do Google Analytics, desenvolvimento de frameworks MapReduce e Hadoop. Conceito de Big Data ganha proeminência.

2010-presente: Machine Learning e AI

Democratização do machine learning, cloud computing, deep learning e inteligência artificial. Surgimento de ferramentas acessíveis como Python/R ecosystems.

Conceitos

Os 4 V's do Big Data

Volume, Velocidade, Variedade e Veracidade

Volume de Dados

Estima-se que 90% dos dados mundiais foram criados nos últimos dois anos. Organizações lidam com petabytes de informações diariamente, exigindo infraestruturas especializadas para armazenamento e processamento.

Velocidade de Processamento

Dados são gerados em tempo real através de sensores IoT, redes sociais, transações financeiras. A capacidade de processar e responder em tempo real tornou-se vantagem competitiva crucial.

Variedade de Formatos

Dados estruturados (bancos relacionais), semi-estruturados (JSON, XML) e não-estruturados (textos, imagens, vídeos) requerem abordagens tecnológicas diversificadas para processamento eficiente.

Veracidade dos Dados

Qualidade, consistência e confiabilidade dos dados são fundamentais. Dados incorretos ou enviesados podem levar a decisões equivocadas com impactos significativos nos negócios.

Unidade I

Engenharia de Dados: ETL

Extract, Transform, Load - A fundação da ciência de dados

A engenharia de dados constitui a fundação sobre a qual toda análise de dados é construída. Tradicionalmente citava-se que 80% do tempo em projetos de ciência de dados era dedicado à preparação de dados, porém pesquisas recentes (Anaconda 2020) indicam entre 45-60%, refletindo melhorias em ferramentas e processos, mas ainda evidenciando a importância crítica desta disciplina.

ETL vs ELT: Paradigmas Modernos

Em arquiteturas modernas, observa-se uma transição para ELT (Extract, Load, Transform), onde a transformação ocorre após o carregamento, aproveitando o poder computacional de data warehouses modernos como Snowflake, BigQuery e Redshift.

Extract (Extração)

Processo de coleta de dados de fontes heterogêneas: APIs REST, bancos de dados relacionais e NoSQL, arquivos CSV/JSON, web scraping, streams de dados em tempo real.

Transform (Transformação)

Aplicação de regras de negócio, limpeza de dados, normalização, agregações, joins complexos, tratamento de valores ausentes e outliers.

Load (Carregamento)

Inserção eficiente em sistemas de destino, considerando estratégias de paralelização, particionamento, indexação e mecanismos de recuperação.

Arquitetura

Modelagem Dimensional

Tabelas Fato e Dimensão

A modelagem dimensional, desenvolvida por Ralph Kimball, permanece fundamental para data warehousing. Esta abordagem organiza dados em fatos (métricas quantitativas) e dimensões (contexto descritivo), otimizando consultas analíticas.

Tabelas Fato

Armazenam métricas de negócio quantificáveis (vendas, cliques, transações). Características: granularidade específica, chaves estrangeiras para dimensões, medidas aditivas e não-aditivas.

Tabelas Dimensão

Fornecem contexto descritivo para análise (cliente, produto, tempo, localização). Implementam hierarquias navegacionais e estratégias de slowly changing dimensions (SCD).

Esquema Estrela vs Esquema Floco de Neve

O esquema estrela oferece simplicidade e performance superior para consultas analíticas, enquanto o esquema floco de neve proporciona normalização mais rigorosa, reduzindo redundância mas aumentando complexidade de consultas.

Compliance

LGPD e Governança

Lei Geral de Proteção de Dados

A LGPD estabelece um marco regulatório rigoroso que impacta significativamente práticas de ciência de dados, exigindo implementação de privacy by design e accountability em todos os processos de tratamento de dados pessoais.

Privacy by Design

Integração de considerações de privacidade desde a concepção de sistemas. Inclui técnicas como differential privacy, federated learning e homomorphic encryption.

Data Subject Rights

Implementação técnica de direitos do titular: acesso, retificação, portabilidade, eliminação. Requer arquiteturas que suportem manipulação granular de dados pessoais.

Impact Assessment

Desenvolvimento de frameworks para avaliação de impacto, especialmente para algoritmos de tomada de decisão automatizada.

Data Lineage

Rastreabilidade completa do ciclo de vida dos dados, desde origem até consumo final. Essencial para auditoria e compliance.

Glossário de Siglas e Termos

OEE

Overall Equipment Effectiveness (Eficácia Geral do Equipamento)
Métrica que mede a eficiência da produção multiplicando três fatores: Disponibilidade (% tempo em operação) × Performance (% velocidade nominal) × Qualidade (% peças boas). Exemplo: 90% × 85% × 98% = 75% OEE.

KPI

Key Performance Indicator (Indicador-Chave de Performance)
Métricas quantificáveis usadas para avaliar o sucesso de processos. Exemplos: produção diária, taxa de refugo, tempo de setup, entregas no prazo.

Setup

Tempo de Preparação
Tempo necessário para preparar uma máquina ou linha de produção ao trocar de um produto para outro. Inclui ajustes, trocas de ferramentas e testes. Quanto menor, mais eficiente a produção.

Refugo

Scrap / Rejeito
Produtos defeituosos que não atendem aos padrões de qualidade e precisam ser descartados ou reprocessados. Medido em porcentagem do total produzido.

Mix de Produção

Product Mix
Combinação ótima de diferentes produtos a serem fabricados considerando restrições de recursos (tempo, matéria-prima, mão de obra) para maximizar lucro ou minimizar custos.

Anomalia

Outlier / Desvio
Comportamento ou valor que se desvia significativamente do padrão esperado. Pode indicar problemas no processo, erros de medição ou eventos especiais que merecem investigação.

Técnicas e Metodologias Utilizadas

1. Análise Exploratória de Dados (EDA)

O que é: Processo de investigação inicial dos dados para descobrir padrões, detectar anomalias e testar hipóteses usando estatísticas descritivas e visualizações.

Como fazemos: Calculamos médias, medianas, desvios padrão; criamos histogramas, boxplots e gráficos de dispersão; identificamos correlações entre variáveis.

Por que importa: Antes de criar modelos complexos, precisamos entender profundamente os dados. 80% do trabalho em ciência de dados é preparação e exploração.

2. Programação Linear (Linear Programming)

O que é: Técnica matemática para encontrar a melhor solução (máximo ou mínimo) de um problema com múltiplas variáveis e restrições lineares.

Como fazemos: Definimos uma função objetivo (ex: maximizar lucro), variáveis de decisão (quantidades a produzir) e restrições (limites de recursos). O algoritmo encontra a solução ótima.

Aplicações: Otimização de mix de produtos, programação de produção, roteamento de veículos, alocação de recursos, planejamento de cortes.

3. Séries Temporais (Time Series)

O que é: Conjunto de dados coletados em intervalos regulares ao longo do tempo. Permite analisar tendências, sazonalidade e fazer previsões.

Como fazemos: Usamos o algoritmo Prophet que automaticamente detecta padrões diários, semanais e anuais, ajusta-se a feriados e anomalias, e projeta valores futuros com intervalos de confiança.

Aplicações: Previsão de demanda, planejamento de produção, gestão de estoque, análise de vendas.

4. Detecção de Anomalias

O que é: Identificação automática de observações que diferem significativamente do comportamento normal esperado.

Métodos utilizados:

  • Z-Score: Mede quantos desvios padrão um valor está da média. Valores com |Z| > 3 são considerados anômalos.
  • IQR (Intervalo Interquartil): Usa a diferença entre o 3º e 1º quartis. Valores fora de 1.5×IQR são outliers.
  • Isolation Forest: Algoritmo de machine learning que isola anomalias construindo árvores de decisão aleatórias.

Aplicações: Detecção de falhas em equipamentos, identificação de fraudes, controle de qualidade, monitoramento de processos.

5. Visualização de Dados

O que é: Representação gráfica de informações para facilitar compreensão, identificação de padrões e comunicação de insights.

Bibliotecas usadas:

  • Matplotlib: Biblioteca fundamental para gráficos estáticos. Altamente customizável, ideal para publicações científicas.
  • Seaborn: Construída sobre Matplotlib, oferece estilos mais modernos e facilita gráficos estatísticos complexos.
  • Plotly: Cria gráficos interativos (zoom, hover, filtros). Ideal para dashboards e apresentações executivas.

Bibliotecas Python Utilizadas

Pandas

Biblioteca fundamental para manipulação de dados estruturados em formato de tabelas (DataFrames). Permite carregar, limpar, transformar e analisar dados de forma eficiente. Pense nela como uma planilha Excel turbinada com superpoderes de programação.

NumPy

Base para computação numérica em Python. Fornece arrays multidimensionais e funções matemáticas otimizadas. É a fundação sobre a qual pandas, scikit-learn e outras bibliotecas científicas são construídas.

Scikit-learn

Principal biblioteca de machine learning em Python. Oferece algoritmos para classificação, regressão, clustering e pré-processamento de dados. Interface consistente e bem documentada, ideal para aprender ML.

Prophet

Desenvolvida pelo Facebook (Meta) especificamente para previsão de séries temporais de negócios. Lida automaticamente com tendências, sazonalidade múltipla, feriados e dados faltantes. Muito robusto para dados reais "bagunçados".

PuLP

Framework para modelagem de problemas de programação linear. Permite definir problemas de otimização de forma intuitiva e resolve usando algoritmos industriais. Amplamente usado em pesquisa operacional.

SciPy

Biblioteca para computação científica avançada. Inclui otimização, integração numérica, álgebra linear, estatística e processamento de sinais. Complementa NumPy com algoritmos mais especializados.

4Otimização Linear: Mix de Produção

Resolver problema clássico de otimização de mix de produtos usando programação linear.

Problema de Mix de Produção

Uma fábrica produz 3 produtos diferentes com recursos limitados. Queremos maximizar o lucro total respeitando as restrições de capacidade, demanda e matéria-prima.

Otimização com PuLP 
# Dados do problema
produtos = ['Produto_A', 'Produto_B', 'Produto_C']
lucro = {'Produto_A': 40, 'Produto_B': 30, 'Produto_C': 50}
tempo_producao = {'Produto_A': 2, 'Produto_B': 1, 'Produto_C': 3}
materia_prima = {'Produto_A': 3, 'Produto_B': 2, 'Produto_C': 4}
demanda_max = {'Produto_A': 200, 'Produto_B': 300, 'Produto_C': 150}

# Recursos disponíveis
horas_disponiveis = 600
materia_prima_disponivel = 800

# Criar problema de otimização
prob = LpProblem("Mix_Producao", LpMaximize)

# Variáveis de decisão
x = LpVariable.dicts("Quantidade", produtos, lowBound=0, cat='Integer')

# Função objetivo: Maximizar lucro
prob += lpSum([lucro[i] * x[i] for i in produtos]), "Lucro_Total"

# Restrições
prob += lpSum([tempo_producao[i] * x[i] for i in produtos]) <= horas_disponiveis, "Tempo"
prob += lpSum([materia_prima[i] * x[i] for i in produtos]) <= materia_prima_disponivel, "Materia_Prima"

for i in produtos:
    prob += x[i] <= demanda_max[i], f"Demanda_{i}"

# Resolver
prob.solve()

# Resultados
print(f"Status: {LpStatus[prob.status]}")
print(f"\nLucro Máximo: R$ {value(prob.objective):,.2f}")
print("\nQuantidades Ótimas:")
for i in produtos:
    print(f"  {i}: {x[i].varValue:.0f} unidades")

# Análise de utilização de recursos
print("\nUtilização de Recursos:")
tempo_usado = sum([tempo_producao[i] * x[i].varValue for i in produtos])
mp_usado = sum([materia_prima[i] * x[i].varValue for i in produtos])
print(f"  Tempo: {tempo_usado:.0f}/{horas_disponiveis} horas ({tempo_usado/horas_disponiveis*100:.1f}%)")
print(f"  Matéria-Prima: {mp_usado:.0f}/{materia_prima_disponivel} kg ({mp_usado/materia_prima_disponivel*100:.1f}%)")

6Detecção de Anomalias em Processos

Identificar automaticamente comportamentos anormais na produção usando técnicas estatísticas.

Detecção de Anomalias 
# Método 1: Z-Score (desvios padrão)
def detectar_anomalias_zscore(df, coluna, threshold=3):
    media = df[coluna].mean()
    desvio = df[coluna].std()
    z_scores = np.abs((df[coluna] - media) / desvio)
    return z_scores > threshold

# Método 2: IQR (Intervalo Interquartil)
def detectar_anomalias_iqr(df, coluna):
    Q1 = df[coluna].quantile(0.25)
    Q3 = df[coluna].quantile(0.75)
    IQR = Q3 - Q1
    limite_inferior = Q1 - 1.5 * IQR
    limite_superior = Q3 + 1.5 * IQR
    return (df[coluna] < limite_inferior) | (df[coluna] > limite_superior)

# Método 3: Isolation Forest (Machine Learning)
from sklearn.ensemble import IsolationForest

def detectar_anomalias_ml(df, colunas):
    modelo = IsolationForest(contamination=0.05, random_state=42)
    predicoes = modelo.fit_predict(df[colunas])
    return predicoes == -1

# Aplicar métodos
df_producao['anomalia_zscore'] = detectar_anomalias_zscore(df_producao, 'producao_unidades')
df_producao['anomalia_iqr'] = detectar_anomalias_iqr(df_producao, 'producao_unidades')
df_producao['anomalia_ml'] = detectar_anomalias_ml(
    df_producao, 
    ['producao_unidades', 'oee', 'refugo_pct']
)

# Visualizar anomalias
fig, ax = plt.subplots(figsize=(16, 6))
ax.plot(df_producao['data'], df_producao['producao_unidades'], 
        label='Produção Normal', alpha=0.7)

# Marcar anomalias
anomalias_ml = df_producao[df_producao['anomalia_ml']]
ax.scatter(anomalias_ml['data'], anomalias_ml['producao_unidades'],
          color='red', s=100, label='Anomalias Detectadas', zorder=5)

ax.set_title('Detecção de Anomalias na Produção', fontsize=14, fontweight='bold')
ax.set_xlabel('Data')
ax.set_ylabel('Unidades Produzidas')
ax.legend()
plt.tight_layout()
plt.show()

print(f"\nAnomalias detectadas:")
print(f"  Z-Score: {df_producao['anomalia_zscore'].sum()} dias")
print(f"  IQR: {df_producao['anomalia_iqr'].sum()} dias")
print(f"  Isolation Forest: {df_producao['anomalia_ml'].sum()} dias")

Boas Práticas de Código

Sempre documente seu código com comentários claros. Use nomes de variáveis descritivos. Organize o código em funções reutilizáveis. Valide seus dados antes de processar. Trate exceções e erros adequadamente. Versione seu código (Git/GitHub).

Resumo da Aula

Nesta aula implementamos soluções práticas em Python para problemas reais da engenharia de produção. Trabalhamos com análise de dados, otimização, previsão e detecção de anomalias usando bibliotecas modernas e atuais. O domínio dessas ferramentas permite automatizar análises complexas e tomar decisões baseadas em dados.
